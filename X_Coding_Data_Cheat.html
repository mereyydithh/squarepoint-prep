PYTHON OOP — ONE-LINERS

Concept One-liner
Encapsulation Bundle data and methods together and HIDE internal state — callers use the public interface, they can't touch the guts.
Inheritance Child classes get parent behaviour for free and can override it — Equity IS-A Instrument, so it automatically has ticker, quantity, price and only needs to implement what's different.
Polymorphism One function works on many types — portfolio_risk_summary() calls .market_value() on every item in the list and doesn't care whether it's an Equity, Bond, or Option.
Abstraction Define WHAT a class must do (interface) without saying HOW — Instrument declares market_value() as @abstractmethod; you can't instantiate it until you've implemented the method.
Composition Prefer "has-a" over "is-a" — Portfolio HAS a list of Instrument objects; it doesn't inherit from Instrument. Looser coupling, easier to change.
@property A method that looks like an attribute — limit.utilization computes on the fly, always consistent, no risk of stale stored state.
@staticmethod A utility function that lives on the class but needs no instance or class state — Instrument.from_bloomberg_ticker("AAPL US").
@classmethod An alternative constructor that receives the class as first arg — Position.from_csv_row(row) lets you build objects from different data sources.
__repr__ What you see in the debugger — implement it so positions print as Equity(AAPL, qty=1000, px=185) not <object at 0x...>.
__eq__ / __hash__ Required to put objects in sets or use them as dict keys — define equality by ticker so two Equity("AAPL", ...) objects match.

SOLID — One Line Each

Principle One-liner
Single Responsibility One class = one reason to change — RiskCalculator computes risk; it does NOT format reports.
Open/Closed Open for extension, closed for modification — add CDS(Instrument) without touching Equity.
Liskov Substitution A child must be usable wherever its parent is — if code takes Instrument, passing an Option must work correctly.
Interface Segregation Don't force a class to implement methods it doesn't need — a thin Priceable interface beats one giant FullInstrument interface.
Dependency Inversion Depend on abstractions, not concretions — RiskEngine depends on the RiskModel protocol, not HistoricalVaRModel directly.

SQL — KEY PATTERNS

-- Window: cumulative P&L
SUM(pnl) OVER (PARTITION BY pm ORDER BY date) AS cum_pnl

-- Window: rolling vol (STDDEV)
STDDEV(ret) OVER (PARTITION BY ticker ORDER BY date ROWS 19 PRECEDING)

-- LAG for daily returns
(close - LAG(close) OVER (ORDER BY date)) / LAG(close) OVER (ORDER BY date)

-- CTE for PM ranking
WITH pm_stats AS (SELECT pm, AVG(pnl) avg, STDDEV(pnl) vol FROM daily GROUP BY pm)
SELECT pm, avg/NULLIF(vol,0)*SQRT(252) sharpe FROM pm_stats ORDER BY sharpe DESC;

-- HHI concentration
SUM(POWER(abs_mv / total_mv, 2)) AS hhi

JOIN rules: INNER = both match | LEFT = keep all left | WHERE = before GROUP | HAVING = after GROUP

DESIGN PATTERNS

Pattern One-liner definition Risk use case
Strategy Define a family of algorithms, encapsulate each one, and make them interchangeable at runtime without changing the calling code. Swap VaR method (hist/param/MC) at runtime — same engine.compute() call, different model injected.
Observer When one object changes state, all registered dependents are automatically notified — publisher/subscriber without tight coupling. Limit breach → automatically triggers email alert + dashboard update + audit log.
Factory Centralise object creation logic so callers don't need to know which concrete class to instantiate. InstrumentFactory.create("equity", row) returns the right class based on instrument type from the DB.
Decorator Wrap a function or class to add behaviour (logging, caching, validation) without modifying its source code. @timer measures execution, @validate_returns checks for NaNs — both stack on top of compute_var().
Pipeline Chain a sequence of processing steps where each step's output is the next step's input. Load → Clean → Calculate → Report — each stage is independent and testable.
Singleton Ensure a class has only one instance and provide a global access point to it. Market data cache, config manager — you want one shared source of truth, not multiple copies.

TESTING — KEY CHECKS

  1. Known distribution: Normal returns → VaR ≈ z × σ × portfolio_value
  2. Monotonicity: Higher vol → higher VaR
  3. ES ≥ VaR: Always, by definition
  4. Directional: Long equity loses in sell-off stress test
  5. Edge cases: Empty series, NaN handling
