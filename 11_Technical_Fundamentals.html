The Four Pillars of OOP

1. Encapsulation
Bundling data and methods together, hiding internal state from outside access.

class RiskLimit:
    def __init__(self, max_var: float):
        self.__max_var = max_var        # private attribute (name-mangled)
        self.__current_var = 0.0

    def update_var(self, new_var: float):
        self.__current_var = new_var

    def is_breached(self) -> bool:
        return self.__current_var > self.__max_var

    @property
    def utilization(self) -> float:
        return self.__current_var / self.__max_var

Why it matters: Internal state (__max_var) can't be accidentally modified from outside. Changes go through controlled methods. In risk systems, this prevents someone from bypassing limit checks.

2. Inheritance
A child class inherits attributes and methods from a parent class, then extends or overrides them.

class Instrument:
    def __init__(self, ticker: str, notional: float):
        self.ticker = ticker
        self.notional = notional

    def market_value(self) -> float:
        raise NotImplementedError    # force subclasses to implement

class Equity(Instrument):
    def __init__(self, ticker: str, shares: int, price: float):
        super().__init__(ticker, shares * price)
        self.shares = shares
        self.price = price

    def market_value(self) -> float:
        return self.shares * self.price

class Bond(Instrument):
    def __init__(self, ticker: str, face: float, clean_price: float, accrued: float):
        super().__init__(ticker, face)
        self.clean_price = clean_price
        self.accrued = accrued

    def market_value(self) -> float:
        return self.face * (self.clean_price + self.accrued) / 100

3. Polymorphism
Different classes can be used through the same interface. The method call is the same, but behavior differs based on the object type.

portfolio = [Equity("AAPL", 1000, 220), Bond("UST10Y", 1_000_000, 98.5, 0.3)]

total_mv = sum(inst.market_value() for inst in portfolio)  # works for any Instrument

Both Equity and Bond have .market_value() but compute it differently. Code that calls .market_value() doesn't need to know the instrument type.

4. Abstraction
Hiding complex implementation details behind a simple interface. Users interact with what something does, not how.

from abc import ABC, abstractmethod

class RiskModel(ABC):
    @abstractmethod
    def compute_var(self, positions, confidence: float) -> float:
        """Compute Value-at-Risk."""
        pass

class HistoricalVaR(RiskModel):
    def compute_var(self, positions, confidence=0.99):
        returns = self._get_historical_returns(positions)
        return -np.percentile(returns, (1 - confidence) * 100)

class MonteCarloVaR(RiskModel):
    def compute_var(self, positions, confidence=0.99):
        simulated = self._run_simulations(positions, n=10000)
        return -np.percentile(simulated, (1 - confidence) * 100)

The caller just does model.compute_var(positions) — doesn't care if it's historical or Monte Carlo underneath.

FROM positions

FROM positions p
INNER JOIN market_data m ON p.ticker = m.ticker
WHERE p.portfolio = 'PM_SMITH';

FROM positions p
LEFT JOIN market_data m ON p.ticker = m.ticker;

-- Window functions: Running total, rank
SELECT ticker, trade_date, pnl,
       SUM(pnl) OVER (PARTITION BY ticker ORDER BY trade_date) as cumulative_pnl,
       RANK() OVER (ORDER BY pnl DESC) as pnl_rank
FROM daily_pnl
WHERE trade_date >= '2026-04-01';

-- Subquery: Positions exceeding VaR limit
SELECT pm_name, var_amount
FROM pm_risk_summary
WHERE var_amount > (SELECT var_limit FROM risk_limits WHERE limit_type = 'PM_VAR');

-- CTE (Common Table Expression): Cleaner complex queries
WITH daily_returns AS (
    SELECT ticker, trade_date,
           (close - LAG(close) OVER (PARTITION BY ticker ORDER BY trade_date))
           / LAG(close) OVER (PARTITION BY ticker ORDER BY trade_date) as ret
    FROM prices
)
SELECT ticker, STDDEV(ret) * SQRT(252) as annual_vol
FROM daily_returns
GROUP BY ticker;

Fat Tails — Why Normal Isn't Enough

Financial returns are NOT normally distributed:
- Excess kurtosis: More extreme events than the normal distribution predicts
- A "once in 10,000 years" event under normal assumptions happens roughly every 10-20 years in reality
- Skewness: Equity returns are left-skewed (crashes are bigger than rallies)
- This is why VaR with normal assumptions underestimates risk — and why Expected Shortfall (ES) is preferred

4.2 Key Statistical Concepts

Bayes' Theorem

$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)}$$

Interview classic: "A test for a disease is 99% accurate. The disease affects 1 in 10,000 people. If you test positive, what's the probability you actually have the disease?"

$$P(\text{disease}|\text{positive}) = \frac{0.99 \times 0.0001}{0.99 \times 0.0001 + 0.01 \times 0.9999} = \frac{0.000099}{0.000099 + 0.009999} \approx 0.98\%$$

Even with a 99% accurate test, you have less than 1% chance of actually having the disease. Base rate matters.

Finance application: "A PM has a track record of 70% winning days. On winning days, they trade 90% of the time. On losing days, they trade 50% of the time. If they traded today, what's the probability it was a winning day?"

$$P(\text{win}|\text{traded}) = \frac{0.90 \times 0.70}{0.90 \times 0.70 + 0.50 \times 0.30} = \frac{0.63}{0.63 + 0.15} = 80.8\%$$

Hypothesis Testing

Step What You Do
1. State hypotheses $H_0$ (null): no effect. $H_1$ (alternative): there IS an effect.
2. Choose significance level $\alpha$ = 0.05 (5%) or 0.01 (1%). Probability of rejecting $H_0$ when it's true (Type I error).
3. Compute test statistic t-stat, z-stat, chi-squared, depending on the test.
4. Compare to critical value or p-value If p-value < $\alpha$, reject $H_0$.

Type I error (False Positive): Rejecting $H_0$ when it's true. "Saying a PM has alpha when they don't."
Type II error (False Negative): Failing to reject $H_0$ when $H_1$ is true. "Saying a PM has no alpha when they do."

t-test for a PM's alpha:

$$t = \frac{\bar{r} - 0}{s / \sqrt{n}}$$

Where $\bar{r}$ = average daily return, $s$ = standard deviation, $n$ = number of days.

If the PM averaged 5bps/day with 80bps daily vol over 252 days:

$$t = \frac{0.0005}{0.008 / \sqrt{252}} = \frac{0.0005}{0.000504} = 0.99$$

A t-stat of 0.99 is NOT statistically significant (need ~2.0 at 5% level). 252 days of 5bps/day with 80bps vol is indistinguishable from luck. This is why multi-year track records matter.

Regression (OLS)

$$y_i = \beta_0 + \beta_1 x_i + \epsilon_i$$

Concept Definition Finance Application
$\beta_1$ Slope — the sensitivity of $y$ to $x$ Beta of a stock to the market
$R^2$ Fraction of variance explained by the model (0 to 1) How much of your PM's returns are explained by market moves
Adjusted $R^2$ $R^2$ penalized for number of predictors Prevents overfitting — adding variables always increases $R^2$, adjusted $R^2$ can decrease
p-value of $\beta$ Probability that $\beta$ is zero (no relationship) Is the factor exposure statistically significant?
Multicollinearity Independent variables are correlated with each other Including both oil AND gasoline as regressors — they're ~0.95 correlated. Coefficients become unstable. Check VIF (Variance Inflation Factor).
Heteroscedasticity Variance of residuals isn't constant Financial data almost always has this (vol clustering). Use robust standard errors (White/Newey-West).
Autocorrelation Residuals are correlated across time Time-series data — today's return is influenced by yesterday's. Durbin-Watson test. Newey-West standard errors.

Multiple regression — Factor model:

$$r_{\text{portfolio}} = \alpha + \beta_1 r_{\text{market}} + \beta_2 r_{\text{rates}} + \beta_3 r_{\text{oil}} + \epsilon$$

This decomposes portfolio returns into: alpha (skill) + factor exposures (beta) + residual (noise). If $\alpha$ is not statistically significant, the PM is not adding value.

Correlation vs Causation

If asked: "Correlation measures linear association between two variables. It does NOT imply one causes the other. Ice cream sales and drowning deaths are correlated (both increase in summer) — the common cause is temperature. In finance, correlation between assets can change dramatically in crisis (correlation breakdown), which is why we stress-test beyond normal correlation assumptions."

Key formulas:

$$\rho_{X,Y} = \frac{\text{Cov}(X,Y)}{\sigma_X \cdot \sigma_Y}$$

$$\text{Cov}(X,Y) = E[(X - \mu_X)(Y - \mu_Y)]$$

Correlation Interpretation
+1.0 Perfect positive — moves together
+0.5 to +0.8 Moderate positive — related sectors
0 No linear relationship
-0.5 to -0.8 Moderate negative — hedging relationship
-1.0 Perfect negative — perfect hedge

Central Limit Theorem (CLT)

The average of a large number of independent random variables will be approximately normally distributed, regardless of the underlying distribution.

This is WHY the normal distribution shows up in finance even though individual returns aren't normal. Portfolio returns (sums of many positions) tend toward normality. But the CLT breaks down when: positions are correlated, the number of positions is small, or distribution tails are extremely fat.

Law of Large Numbers

As sample size increases, the sample mean converges to the population mean.

This is WHY a PM needs a long track record — a few months of returns could be luck. Over many trades/days, true skill (or lack thereof) reveals itself.

Monte Carlo Simulation — The How

import numpy as np

def monte_carlo_var(returns: np.ndarray, portfolio_value: float,
                     confidence: float = 0.99, n_sims: int = 100_000) -> float:
    """
    Parametric Monte Carlo VaR.
    Fits a distribution to historical returns, then simulates.
    """
    mu = returns.mean()
    sigma = returns.std()

    # Simulate future returns
    simulated_returns = np.random.normal(mu, sigma, n_sims)
    simulated_pnl = portfolio_value * simulated_returns

    # VaR is the loss at the confidence percentile
    var = -np.percentile(simulated_pnl, (1 - confidence) * 100)
    return var

For correlated multi-asset portfolios, use Cholesky decomposition:

$$L = \text{cholesky}(\Sigma)$$
$$\text{correlated_returns} = L \cdot z$$

Where $z$ = vector of independent standard normal draws, $\Sigma$ = covariance matrix, $L$ = lower triangular Cholesky factor.

PART 5: BRAINTEASERS & PROBABILITY PUZZLES

Quant fund interviews love these. They test mental math, probability intuition, and how you think under pressure.

5.1 Probability Classics

Q: You flip a fair coin twice. Given that at least one flip is heads, what's the probability both are heads?

Sample space: HH, HT, TH, TT. "At least one head" eliminates TT → {HH, HT, TH}.
$P(\text{both heads} | \text{at least one head}) = 1/3$.
Not 1/2! The conditioning changes the sample space.

Q: You roll two dice. What's the probability the sum is 7?

Total outcomes: 36. Outcomes summing to 7: (1,6), (2,5), (3,4), (4,3), (5,2), (6,1) = 6.
$P = 6/36 = 1/6$.

Q: A drawer has 8 black socks and 6 white socks. How many socks do you need to pull out (in the dark) to guarantee a matching pair?

3 socks. By pigeonhole principle: 2 colors, so 3 socks guarantees at least 2 of one color.

Q: You have 3 cards: one red on both sides, one blue on both sides, one red on one side and blue on the other. You pick a random card and see a red side. What's the probability the other side is also red?

3 red faces in total: 2 are on the red-red card, 1 is on the red-blue card.
$P(\text{other side red} | \text{you see red}) = 2/3$.

Q: What's the expected number of coin flips to get heads?

Geometric distribution. $E[X] = 1/p = 1/0.5 = 2$ flips.

Q: What's the expected number of coin flips to get two heads in a row?

Let $E$ = expected flips. After first H (takes 2 flips on average):
- If next is H → done (1 more flip)
- If next is T → start over (E more flips)

$E = 2 + 0.5 \times 1 + 0.5 \times E$
$0.5E = 2.5$
$E = 6$ flips.

Q: Two trains are 100 miles apart heading toward each other at 50 mph each. A bird flies between them at 75 mph. How far does the bird fly before the trains meet?

Trains close at 100 mph combined → meet in 1 hour. Bird flies for 1 hour at 75 mph → 75 miles.
Don't get tricked into computing infinite series.

5.2 Estimation / Fermi Problems

Q: How many piano tuners are in New York City?

Framework:
- NYC population: ~8.3M
- Households: ~3.2M
- % with pianos: ~5% → 160,000 pianos
- Tunings per year: ~1.5 → 240,000 tunings/year

Q: Find the top 3 drawdown periods for a cumulative P&L series.

def find_drawdowns(cumulative_pnl: pd.Series, top_n: int = 3):
    peak = cumulative_pnl.cummax()
    drawdown = cumulative_pnl - peak
    is_drawdown = drawdown < 0

    # Label consecutive drawdown periods
    groups = (~is_drawdown).cumsum()
    drawdown_groups = drawdown[is_drawdown].groupby(groups)

    worst = drawdown_groups.min().nsmallest(top_n)
    return worst

def compute_vwap(trades: pd.DataFrame) -> pd.Series:
    trades['notional'] = trades['qty'] * trades['price']
    return trades.groupby('ticker')['notional'].sum() / trades.groupby('ticker')['qty'].sum()

6.2 Algorithmic Thinking

Q: Given a sorted list of P&L values, find the pair with the smallest difference.

def min_diff_pair(pnls: list[float]) -> tuple[float, float]:
    """O(n) after sort — just check adjacent elements."""
    pnls.sort()
    min_diff = float('inf')
    result = (0, 0)
    for i in range(len(pnls) - 1):
        diff = pnls[i+1] - pnls[i]
        if diff < min_diff:
            min_diff = diff
            result = (pnls[i], pnls[i+1])
    return result

Q: Implement a function that checks if a string of brackets is balanced.

def is_balanced(s: str) -> bool:
    """Classic stack problem. O(n) time, O(n) space."""
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    for ch in s:
        if ch in '([{':
            stack.append(ch)
        elif ch in ')]}':
            if not stack or stack[-1] != pairs[ch]:
                return False
            stack.pop()
    return len(stack) == 0

6.3 Risk-Specific Coding

Q: Implement a simple VaR calculation using historical simulation.

def historical_var(returns: pd.Series, confidence: float = 0.99,
                   portfolio_value: float = 1_000_000) -> float:
    """
    Historical VaR: sort returns, pick the quantile.
    """
    sorted_returns = returns.sort_values()
    index = int(len(sorted_returns) * (1 - confidence))
    var_return = sorted_returns.iloc[index]
    return -var_return * portfolio_value

# Usage
daily_returns = prices.pct_change().dropna()
var_99 = historical_var(daily_returns, confidence=0.99)
print(f"1-day 99% VaR: ${var_99:,.0f}")

Q: Compute the Sharpe ratio from a return series.

def sharpe_ratio(returns: pd.Series, risk_free_rate: float = 0.0375) -> float:
    """Annualized Sharpe ratio."""
    excess_returns = returns - risk_free_rate / 252  # daily risk-free
    return excess_returns.mean() / excess_returns.std() * np.sqrt(252)

PART 7: QUICK-FIRE DEFINITIONS

If they throw rapid-fire terminology at you:

Term 10-Second Answer
API Application Programming Interface — a contract for how software components communicate. REST APIs use HTTP.
JSON JavaScript Object Notation — text format for structured data. Key-value pairs, nested. Standard for APIs.
Git Version control system. commit, branch, merge, pull request.
Docker Containerization — packages code + dependencies into a portable container. "Works on my machine" → works everywhere.
AWS EC2 Virtual servers in the cloud. Scalable compute.
AWS S3 Object storage in the cloud. Store files (data, models, logs).
Lambda function Anonymous inline function: lambda x: x * 2. One expression.
Map / Filter / Reduce Functional programming. map(f, list) applies f to each element. filter(pred, list) keeps matching elements.
Concurrency vs Parallelism Concurrency: multiple tasks in progress (may alternate). Parallelism: multiple tasks executing simultaneously (multiple CPU cores).
GIL (Global Interpreter Lock) Python limitation — only one thread executes Python bytecode at a time. Use multiprocessing for true parallelism.
Mutable vs Immutable Mutable: can change after creation (list, dict). Immutable: cannot (tuple, string, frozenset).
Stack vs Queue Stack: LIFO (last in, first out). Queue: FIFO (first in, first out).
Hash Table Key-value store with O(1) average lookup. Python dict is a hash table.
Big O Notation Describes algorithm efficiency. O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ).
REST API Architecture for web services. GET (read), POST (create), PUT (update), DELETE (remove). Stateless.
SOLID Design principles: Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion.
DRY "Don't Repeat Yourself" — avoid code duplication.
Race condition Bug where behavior depends on timing of concurrent operations. Two processes writing to the same file.
Deadlock Two processes each waiting for the other to release a resource. Neither can proceed.
