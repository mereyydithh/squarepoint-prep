Volatility Models — Hierarchy

Model Key formula / feature Limitation
Historical vol σ = stdev(returns, N) All obs weighted equally
EWMA σ²ₜ = λσ²ₜ₋₁ + (1-λ)r²ₜ₋₁; λ=0.94 No mean reversion
GARCH(1,1) σ²ₜ = ω + αr²ₜ₋₁ + βσ²ₜ₋₁ Symmetric — ignores leverage effect
GJR-GARCH Adds γ for neg returns → asymmetric More params → calibration risk

GARCH Key Relationships

  • Persistence = α + β (typical: 0.95-0.99)
  • Long-run vol = √(ω / (1 - α - β) × 252)
  • Half-life = -ln(2) / ln(α + β)
  • α + β ≥ 1.0 → IGARCH → vol shocks NEVER decay → suspicious

4-Question Framework (Challenging Quants)

  1. ASSUMPTIONS: Distribution? Correlation structure? Liquidity? Lookback sensitivity?
  2. DATA: Includes crises? Survivorship bias? Look-ahead bias? Outlier handling?
  3. VALIDATION: Out-of-sample R²? VaR backtest breaches? Walk-forward test?
  4. FAILURE MODES: "What would make you lose confidence in this model?"

8 Deadly Sins of Model Risk

  1. Gaussian assumption (normal returns)
  2. Stable correlations (regime-blind)
  3. Infinite liquidity (instant exit)
  4. Stationary parameters (no regime adapt)
  5. Overfitting (noise as signal)
  6. Survivorship bias (winners-only data)
  7. Look-ahead bias (future info leak)
  8. Mark-to-model (favorable internal pricing)

Quick Interview Lines

  • "Models are maps, not territory — use them as info signals while knowing their failure modes"
  • "VaR says $5M → I ask: under what assumptions? What's ES? What's gamma?"
  • "GARCH captures vol clustering + mean reversion; α+β = persistence; prefer GJR for equities (leverage effect)"
  • "Best single question to a quant: 'What would make you lose confidence in this model?'"
  • "Sharpe 3.0 in backtest → expect 1.0-1.5 live. Always check out-of-sample."
